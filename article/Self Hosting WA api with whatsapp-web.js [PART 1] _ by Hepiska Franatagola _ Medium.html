Self Hosting WA api with whatsapp-web.js [PART 1]

Hepiska Franatagola

·

5 min read

One option when u need whatsapp capability on your apps is using wa official business api but to be able to deploy this to prod you have to have landing page that explaining what happen with your apps and get approval, other option is using un official wa service. currently i found a node js library called whatsapp-seb.js where u could build your own wa api. this library run puppeteer and basically run browser and open wa via and listen every message there and forward it to the app and vice versa when u send message from your app.

on this article i will explain how to create your own seof hosting wa api base on whatsapp-web.js on until you can build your own docker image and deploy it. on this project I’m using base repo that i usually use when i start node js project you can clone and use it if you like it basic-crud-express-mongo, okai lets start

Clone the and prepare base project

# go to your target directory 
git clone https://github.com/hepiska/basic-crud-express-mongo.git

This base project write in TS and already included expressjs, mongo db, authentication and the folder structure will look like this.

after we clone the project than run npm i this will install necessary package after that try run the app by run npn run dev currently will throw error because this app need public and private key to handle auth, mongo path to access the db. to solve this :

Go to src/utils/jwt.ts and replace the code inside with this code

Install the whatsapp-web.js

to install whatsapp-web.js we can run npm i -S whatsapp-web.js qrcode-terminal @types/qrcode-terminal and also to display the QR code on terminal sp we can connect our wa number to the api. create new folder inside src folder called wa-messagededicated to all the code regarding this wa-message

the .consumer file will listen all event send by wa there is couple important event that we need to listen

  client.on('qr', (qr) => {
     qrcode.generate(qr, { small: true }) // generate qr code in terminal 
  })

this event trigger when the QR code generated by wa-web in this we need to display the qr code generated to terminal or to web. for this part we will generate to terminal.

  client.on('authenticated', () => {
    logger.info('Client is authenticated!');
  });

this event trigger after we scan the qr code and our wa authenticate and can be use.

  client.on('message', async (message: LocaMessage) => {
    try {
       logger.info("=========message", message)
       } catch (error) {
      logger.error((error as Error).message)
    }
  });

on message this is event that we will listen an use the most because all off the logic will live here

before we can listen all off the event above we need to instantiate there is couple auth options to instantiate the client for for me i will use local auth. the complete code bellow

import { Client, ClientOptions, LocalAuth, Message } from "whatsapp-web.js"
import qrcode from "qrcode-terminal"
import logger from "../utils/logger";


const waNumbers = ['6281234567890', '6281234567891']

const clients: Record<string, Client> = {}

waNumbers.forEach((waNumber) => {
  const clientConfig: ClientOptions = {
    authStrategy: new LocalAuth({
      clientId: waNumber
    })
  }

  const client = new Client(clientConfig)

  client.on('qr', (qr) => {
    qrcode.generate(qr, { small: true })
  })

  client.on('authenticated', (session) => {
    logger.info(`Client ${waNumber} authenticated`)
  })

  client.on('ready', () => {
    logger.info(`Client ${waNumber} ready`)
  })

  client.on('message', (message: Message) => {
    logger.info(`Client ${waNumber} received message: ${message.body}`)
    client.sendMessage(message.from, `You said: ${message.body}`)
  })

  clients[waNumber] = client

  client.initialize()

})

export default clients

On this line const waNumber = ['62812123456', '62012223121']create an array to save the wa numbers that we need to use, and have emty object that later to save the clients so we can have multiple wa in our app. Looping every wa number if you see this code

const clientConfig: ClientOptions = {
    authStrategy: new LocalAuth({
      clientId: waNumber
    })
  }
const client = new Client(clientConfig)

this code we set the configuration and instantiate the client for each wa number that we have and clients[waNumber] = client to save the client created to object we declare before with wa number as reference last we export clients so the client can be use by other logic in other file. and last we import the wa-message.consumer.ts in src/index.ts import “./wa-message/wa-message.consumer”.

after we run the app will the barcode will aprear

and after we scan the barcode we will this our number already autenticated
