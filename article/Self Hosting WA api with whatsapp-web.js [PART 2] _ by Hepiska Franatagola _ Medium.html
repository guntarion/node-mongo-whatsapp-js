create new file in /src called web-socket.ts and pass code bellow this code to instantiate the socket.io and we can useit latter in our code

import { Server } from "socket.io";
import logger from "./utils/logger";


export const initiateScoket = (server: any) => {
  const io = new Server(server, {
    path: "/socket/"
  })

  io.on("connection", (socket) => {
    logger.info("a user connected");
  })

  return io
}

add addjustment in index.js so our express can also serve websocket by socket io. I put the explanation of the change in comment. if all success you will see log user-conencted when u open localhost:3000/app

Set up Docker

Create dockerfile inside root folder and put this script basically this will usr no 18, and install headless chrome to run pupetiers then build the app.

FROM node:18-slim


# Install Google Chrome Stable and fonts
# Note: this installs the necessary libs to make the browser work with Puppeteer.
RUN apt-get update && apt-get install gnupg wget -y && \
  wget --quiet --output-document=- https://dl-ssl.google.com/linux/linux_signing_key.pub | gpg --dearmor > /etc/apt/trusted.gpg.d/google-archive.gpg && \
  sh -c 'echo "deb [arch=amd64] http://dl.google.com/linux/chrome/deb/ stable main" >> /etc/apt/sources.list.d/google.list' && \
  apt-get update && \
  apt-get install google-chrome-stable -y --no-install-recommends && \
  rm -rf /var/lib/apt/lists/*
RUN apt-get update  && apt-get install -y gconf-service libgbm-dev libasound2 libatk1.0-0 libc6 libcairo2 libcups2 libdbus-1-3 libexpat1 libfontconfig1 libgcc1 libgconf-2-4 libgdk-pixbuf2.0-0 libglib2.0-0 libgtk-3-0 libnspr4 libpango-1.0-0 libpangocairo-1.0-0 libstdc++6 libx11-6 libx11-xcb1 libxcb1 libxcomposite1 libxcursor1 libxdamage1 libxext6 libxfixes3 libxi6 libxrandr2 libxrender1 libxss1 libxtst6 ca-certificates fonts-liberation libappindicator1 libnss3 lsb-release xdg-utils


WORKDIR /usr/src/app


COPY package*.json ./

RUN npm ci

COPY . .

RUN npm run build
COPY ./src/public ./dist/public


CMD [ "node", "./dist/index.js" ]

after this we need make some change in our app so it can work smoothly. first if you see our project folder there two folder .wwebs_auth and .wweb_cache and that automatic created after we run our app this two folder save the wa session and cached data for the pupeteers. We need to be able to change this path so later we can move it in the machine volume when we run the docker.

We need to change configuration when we instanciate the wa client by this script

  const clientConfig: ClientOptions = {
    webVersionCache: {
      type: 'local',
      path: config.wa_session_file_path + "/.web_cache"
    },
    authStrategy: new LocalAuth({
      clientId: waNumber,
      dataPath: config.wa_session_file_path
    })
  }

and add edit our src/config/index.ts to and add our desire path on .env by variable WA_SESSION_FILE_PATH.

import dotenv from 'dotenv';
dotenv.config()


export default {
  mongo_uri: process.env.MONGO_URI || "mongodb://localhost:27017/test",
  jwt_expiration_time: process.env.JWT_EXPIRATION_TIME || "1h",
  wa_session_file_path: process.env.WA_SESSION_FILE_PATH || "/tmp"
}

Finally we can build our docker image by run docker build -t <image-name> . and if tour local run min m1 procesor you can add -platform linux/amd64 to docker build --platform linux/amd64 -t image-name . now u have docker image that ready to use on your VPS.
